## v0.2.0 — One MCP, many accounts. Now with fan-out.

Nucleus is a local MCP gateway that lets Claude (and other MCP clients) hold multiple authenticated profiles of the same service simultaneously — prod and staging Supabase, work and personal GitHub — without disconnecting between sessions.

This release is the wedge. Six features that stack into a coherent product:

### The headline: `nucleus_call_plan` (multi-profile fan-out)

> *"compare the row count of the users table between prod and staging supabase"*

That sentence was previously two sequential conversations. Now it's one tool call. `nucleus_find_tool` detects fan-out-shaped intents and emits a `fanout_suggestion`; `nucleus_call_plan` runs the steps in parallel (default 4-wide, capped at 16), merges the results, returns one JSON document per call. Partial failures coexist with successes — one slow upstream doesn't kill the whole plan.

### Sticky resolution + `because:` explanations

Every `nucleus_find_tool` hit now carries a `because` array explaining its rank: `"matched 'sql' in tool name"`, `"matched 'atlas' in alias"`, `"sticky from last call"`. Opaque rankers feel arbitrary — this one shows its work. After each successful dispatch, the gateway remembers the alias used per connector and biases ambiguous future ranking toward it. Process-lifetime only, never persisted.

### Policy gate (`policy.toml`)

Optional rule file gating writes/destructives across every dispatch path:

```toml
[[rule]]
match  = "supabase:atlas"
deny   = ["apply_migration", "delete_branch"]
reason = "atlas is the production project — schema changes go through CI"

[[rule]]
match   = "supabase:atlas"
confirm = ["execute_sql"]
phrase  = "I understand atlas is PRODUCTION"
```

`deny` short-circuits over `confirm`. Confirmation phrases land in the call args and the audit trail — no silent overrides.

### Audit log + `nucleus logs`

Every dispatch writes one JSONL line at `~/.nucleusmcp/audit.log`. Privacy by default: arguments stored as sorted top-level keys + SHA-256 hash, never verbatim (set `NUCLEUSMCP_AUDIT_FULL_ARGS=1` to opt in). Result payloads never logged. Rotates at 10 MiB.

```bash
nucleus logs                                  # last 50 entries pretty-printed
nucleus logs --decision denied                # what did the policy block?
nucleus logs --tool execute_sql --since 1h
nucleus logs --json | jq '.tool'              # pipe into jq
```

### Idle reaper (`--idle-timeout`)

Power-user installs running a dozen profiles no longer pay for a dozen idle subprocesses. Children unused past the timeout get reaped; the next call transparently respawns from a captured closure (env rebuilt at respawn time so credential rotations apply). Default disabled.

```bash
nucleus serve --idle-timeout 15m
```

### `nucleus doctor`

First-run UX. Checks claude CLI on PATH, mcp-remote on PATH, registry reachable, ≥1 profile registered, `policy.toml` parses, audit dir writable, connectors load. Optional `--probe` against a running HTTP gateway. PASS/WARN/FAIL with `fix:` hints, `--strict` for CI.

```bash
nucleus doctor
nucleus doctor --probe http://127.0.0.1:8787/mcp
```


Pre-built binaries for macOS / Linux / Windows attached to this release.

## Migration from v0.1.x

No migration needed for typical setups. The on-disk paths (`~/.nucleusmcp/registry.db`, `~/.nucleusmcp/oauth/...`) and OS keychain service name are unchanged. To pick up the new defaults:

1. `brew upgrade nucleus` (or `go install ...@latest` / re-download the binary).
2. Restart any running `nucleus serve` instance.
3. `nucleus doctor` to confirm everything is wired.

If you build against the Go module: the router `makeHandler` signature changed (now takes a `Dispatcher` interface rather than `*supervisor.Child`). Internal API only — no public surface affected.

## Test footprint

86 tests across 6 packages, all green under `-race`. New: 6 router-level integration tests covering the full `find_tool → fanout → call_plan → audit → policy → sticky` chain.
